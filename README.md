# CAN Bus Network Analyzer
### STM32F103 + MCP2515 — Complete Project

---

## Overview

This project implements a complete two-node CAN Bus network using two STM32F103 "Blue Pill" boards and two MCP2515 modules:

| Board | Role | What it does |
|---|---|---|
| STM32 #1 | **ECU Simulator** | Generates fake Speed, RPM, Fuel, Temp, Battery, Throttle data and broadcasts them as CAN frames |
| STM32 #2 | **CAN Analyzer** | Receives every frame, decodes it, checks for faults, logs it, and streams results to your PC via UART |

A Python dashboard (`can_dashboard.py`) runs on your PC and renders a live terminal display of the decoded signals.

> Check yours and update `mcp2515.c` accordingly — see the comment in `MCP2515_Init()`.

## Wiring Diagram

### Each STM32 ↔ MCP2515 connection (identical for both boards)

```
MCP2515 Module        STM32F103 "Blue Pill"
──────────────        ─────────────────────
VCC           ───→    3.3V
GND           ───→    GND
SCK           ───→    PA5   (SPI1 Clock)
SI  / MOSI    ───→    PA7   (SPI1 MOSI)
SO  / MISO    ───→    PA6   (SPI1 MISO)
CS            ───→    PA4   (GPIO output)
INT           ───→    PB0   (optional, not used in this firmware)
```

### CAN Bus wiring

Connect the two MCP2515 modules together:

```
MCP2515 #1          MCP2515 #2
──────────          ──────────
CAN H  ────────────  CAN H
CAN L  ────────────  CAN L
  |                    |
 120Ω                120Ω
  |                    |
 GND                  GND
```

> Both ends of a CAN bus need a **120 Ω termination resistor** between CAN H and CAN L.
> Most MCP2515 modules have a 120 Ω resistor you can enable with a solder bridge or jumper.

### UART connection (Analyzer → PC)

```
STM32 #2 (Analyzer)        USB-TTL Adapter
───────────────────        ───────────────
PA9  (TX)          ───→   RX
PA10 (RX)          ───→   TX   (optional, for sending commands)
GND                ───→   GND
```

## CAN Message Protocol

| CAN ID | Signal | Encoding | Range |
|---|---|---|---|
| `0x100` | Speed | 1 byte, km/h | 0–255 |
| `0x101` | RPM | 2 bytes, big-endian | 0–8000 |
| `0x102` | Fuel | 1 byte, % | 0–100 |
| `0x103` | Coolant Temp | 1 byte, °C | 0–150 |
| `0x104` | Battery Voltage | 2 bytes, mV big-endian | 10000–15000 |
| `0x105` | Throttle | 1 byte, % | 0–100 |
| `0x7DF` | Fault | byte[0]=count, byte[1]=code | — |

## Building with STM32CubeIDE

1. **Open STM32CubeIDE** → File → New → STM32 Project
2. Select **STM32F103C8T6** (or C6 if your board has the smaller flash)
3. Add all `.c` files from `Core/Src/` to the project
4. Add `Core/Inc/` to the include path (Project → Properties → C/C++ Build → Settings → Include Paths)
5. Make sure **HAL drivers** are included (CubeIDE does this by default)
6. Build and flash

> The HAL library and CMSIS files are generated by CubeIDE from the chip selection.
> This project does **not** include them to keep the folder lightweight.

### Alternative: Makefile / PlatformIO

If you use PlatformIO (VS Code), create a `platformio.ini`:

```ini
[env:bluepill_f103c8]
platform      = ststm32
board         = bluepill_f103c8
framework     = stm32cube
build_flags   = -ICore/Inc
src_dir       = Core/Src
```

## Running the PC Dashboard

```bash
pip install pyserial colorama

# Auto-detect port
python can_dashboard.py

# Specify port + save CSV log
python can_dashboard.py --port /dev/ttyUSB0 --log
```

### Dashboard commands (type in terminal while dashboard runs)

| Key | Action |
|---|---|
| `L` | Dump the full in-RAM log |
| `C` | Clear log and reset stats |
| `R` | Toggle raw hex ↔ decoded display |
| `F` | Show current fault status |
| `S` | Show per-ID statistics |
| `H` | Show help |

## Extending the Project

- **OLED display:** Use an SSD1306 I2C OLED (PA1=SDA, PA2=SCL) and display top 4 signals. Library: `ssd1306.h` (many open-source versions available)
- **SD card logging:** SPI SD card module on SPI2 (PB13/PB14/PB15, CS=PB12) for persistent logs
- **Real DBC file decoding:** Automotive `.dbc` files define the full signal encoding of real vehicles. Parse on the PC side in Python using `cantools`
- **Real vehicle connection:** Add an OBD-II to CAN adapter (DB9 connector) and listen to your actual car's bus

## Troubleshooting

| Problem | Likely cause | Fix |
|---|---|---|
| MCP2515 init fails | Wrong crystal frequency | Check CNF registers in `mcp2515.c` |
| No frames received | Bus not terminated | Add 120 Ω between CAN H and CAN L |
| Garbled UART | Wrong baud rate | Confirm 115200 in your serial monitor |
| Only one frame received | CS pin float | Pull CS high with 10 kΩ resistor |
| SPI fails | VCC too low | MCP2515 needs 2.7–5.5 V; some boards need 5V |
